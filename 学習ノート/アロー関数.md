## ① アロー関数（React視点）

### まず結論

👉 **「短く書けて、thisでハマらない関数」**  
Reactでは「イベント処理」「propsで渡す関数」にほぼこれを使う。

---

### 普通の関数との比較

```javascript
// 従来の関数
function add(a, b) {
  return a + b;
}

// アロー関数
const add = (a, b) => {
  return a + b;
}
```

さらに省略すると👇
```javascript
const add = (a, b) => a + b;
```

---

### Reactでよく見る形
```javascript
<button onClick={() => {   console.log("クリックされた"); }}>   クリック </button>
```

これはつまり：
- 「クリックされた時に実行する**処理の塊**」を
- **その場で定義して渡している**
という意味。
---
### なぜReactではアロー関数が多い？

#### 理由①：`this`で事故らない
昔のReact（classコンポーネント）では👇が地獄だった
```javascript
this.handleClick = this.handleClick.bind(this);
```

アロー関数は **thisを自動で固定**してくれる  
→ 今は「考えなくていい」から採用されている

---
#### 理由②：「その場で処理を書く」用途に向いている
Reactは
- ボタンが押されたら
- 入力されたら
- 状態が変わったら
という **小さい処理の集合体**。
アロー関数は  
👉 **「今ここで使う処理」**を書くのに向いている。
